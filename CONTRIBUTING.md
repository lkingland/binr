# CONTRIBUTING

## Building

Run `make` when first downloading the repository.  This will ensure the
test binary server, necessary for tests, is created in ./testdata.

## Issues

Please check Issues for known bugs, requests for help, and good first issues.

## PRs

Please open an issue to ask questions or get clarification before investing
a lot of time in a PR.

Run `make` before submitting a PR to run code quality tools.
